# Vue3 代码审查技能

## 介绍
这是一个专门用于审查 Vue3 项目代码质量的技能。它可以检查常见的代码问题、性能问题、安全问题和最佳实践。

## 功能特性

### 1. 组件结构审查
- 检查组件文件长度（推荐不超过300行）
- 确保包含必需的标签（template、script）
- 验证组件命名规范

### 2. Composition API 审查
- 检查 ref/reactive 的使用
- 验证副作用清理
- 检查组合式函数的组织

### 3. TypeScript 类型安全审查
- 验证是否使用了 TypeScript
- 检查 Props 的类型定义
- 确保 API 响应数据的类型定义

### 4. 性能问题审查
- 检查 v-if 和 v-for 的不当使用
- 识别可能导致性能问题的深度监听
- 检查不必要的计算属性依赖

### 5. 安全问题审查
- 识别使用 v-html 的潜在 XSS 风险
- 检查可能的代码注入漏洞

### 6. 最佳实践审查
- 组件单一职责原则
- 合理的嵌套层级
- 正确的生命周期钩子使用

## 使用方法

当您运行 `/review-frontend` 命令时，如果检测到 Vue3 项目，此技能将自动被调用。

或者直接运行：
```bash
# 在项目根目录运行
claude run vue3-code-review
```

## 输出格式

审查工具会生成一个详细的报告，包括：
- 总体统计信息
- 按严重程度分类的问题
- 具体问题的位置和说明

## 配置选项

- `includePerformance`: 是否检查性能问题（默认 true）
- `includeSecurity`: 是否检查安全问题（默认 true）
- `includeBestPractices`: 是否检查最佳实践（默认 true）

## 严重程度等级

- `critical`: 严重问题，可能导致应用崩溃或安全漏洞
- `high`: 高危问题，可能影响功能或性能
- `medium`: 中等问题，可能影响可维护性
- `low`: 低等问题，建议改进但不影响功能

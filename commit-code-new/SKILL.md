# commit-code-new 技能

## 概述

commit-code-new 是一个智能代码提交和分支管理助手，完全基于Bash实现。它可以帮助用户总结Git更改区的代码变更，自动生成符合约定式提交（Conventional Commits）规范的提交信息，并提供完整的分支管理功能。该技能不仅能识别新增文件和更新文件，还能创建、切换和管理分支，帮助开发者高效地进行代码提交和版本控制。

## 触发条件

当用户表达以下意图时会自动触发此技能：

- "提交我的更改" / "提交代码"
- "生成提交信息" / "commit message"
- "总结代码变更" / "整理 Git 更改"
- "查看 Git 状态" / "准备 Git 提交"
- "写提交信息"
- "创建新分支" / "新建分支"
- "切换分支" / "切换到某个分支"
- "分支管理" / "管理分支"
- "列出分支" / "查看分支"
- 或任何与 Git 提交或分支管理相关的请求

## 功能特性

### 1. Git 更改区域分析

- 自动扫描工作目录中的所有更改
- 区分已暂存和未暂存的更改
- 识别新增、修改、删除和重命名的文件
- 提取更改的关键信息进行摘要

### 2. 文件类型识别

- 自动识别文件类型（新文件、更新文件、删除文件等）
- 根据文件扩展名确定编程语言
- 为不同类型的文件应用合适的处理策略

### 3. 智能提交信息生成

- 根据变更内容自动生成符合 Conventional Commits 规范的提交信息
- 支持 feat、fix、docs、style、refactor、test、chore 等类型
- 提供清晰的提交信息结构（type(scope): description）

### 4. 变更摘要功能

- 提供简洁明了的变更列表
- 突出显示重要的变更部分
- 支持按文件类型或变更类型分类展示

### 5. 安全检查

- 在执行提交前进行预览
- 允许用户编辑生成的提交信息
- 避免意外提交敏感信息

### 6. 分支管理功能

- 创建新分支：根据用户输入的分支名称创建新分支
- 支持从当前分支或指定基础分支创建
- 自动切换到新创建的分支
- 列出所有本地分支和远程分支
- 切换到已有分支
- 分支命名规范检查（如 feature/xxx、bugfix/xxx、hotfix/xxx）
- 提供分支创建建议（基于当前工作内容）

### 7. 推送提醒功能

- 提交成功后自动检查是否有远程仓库
- 检测本地分支是否领先远程分支
- 自动询问用户是否要推送到远程
- 支持推送前的确认预览

## 使用方法

### 基本用法

1. 用户完成代码修改后，调用此技能
2. 技能自动分析当前 Git 状态
3. 生成变更摘要
4. 根据变更内容建议合适的提交信息
5. 用户可以确认或修改提交信息
6. 执行实际的 Git 提交操作
7. 提交成功后，检查是否有远程仓库以及本地是否领先远程
8. 如果有远程仓库且本地领先，自动询问用户是否要推送到远程

### 高级功能

- 支持选择性提交（仅提交部分更改）
- 提供多级别的提交信息详细程度
- 支持自定义提交信息模板
- 验证提交信息是否符合规范

### 分支管理操作

#### 创建新分支

当用户请求创建新分支时，执行以下步骤：

1. **确认基础分支**：使用当前分支作为基础
2. **收集分支名称**：
   - 如果用户提供了分支名称，验证命名规范
   - 如果用户未提供，可基于当前工作内容建议分支名称
3. **分支命名规范**：
   - 新功能：`feature/功能描述` 或 `feat/功能描述`
   - 修复 Bug：`bugfix/问题描述` 或 `fix/问题描述`
   - 紧急修复：`hotfix/问题描述`
   - 发布版本：`release/版本号`
   - 文档更新：`docs/文档描述`
   - 重构：`refactor/重构描述`
4. **执行创建**：使用 `git checkout -b <分支名>` 创建并切换到新分支
5. **确认结果**：显示创建成功的分支信息

#### 切换分支

当用户请求切换分支时：

1. 列出所有本地分支，标记当前分支
2. 支持分支名称匹配
3. 检查当前分支是否有未提交的更改，提醒用户
4. 执行切换操作并确认结果

#### 列出分支

显示所有本地分支和远程分支，用 `*` 标记当前分支

## 适用场景

- 日常代码提交工作流
- 需要规范化提交信息的团队项目
- 提高代码提交效率的个人开发
- 遵循语义化版本控制的项目
- 需要自动化提交过程的持续集成环境

## 注意事项

- 此技能仅在 Git 仓库中运行
- 会尊重用户的 .gitignore 配置
- 执行提交前会提供预览以供确认
- 严格按照 Conventional Commits 规范生成提交信息
- 完全基于Bash实现，无需Python环境

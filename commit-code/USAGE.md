# commit-code 技能使用说明

## 概述

commit-code 是一个智能化的 Git 提交助手，它能够分析您的代码更改，自动总结变更内容，并生成符合 Conventional Commits 规范的提交信息。此外，它还提供全面的仓库管理功能，包括初始化、撤销初始化、删除仓库等操作。

## 安装和配置

这个技能已经内置在您的 Claude Code 环境中，无需额外安装。

## 基本使用

### 1. 准备更改
在使用此技能之前，请确保您已经：
- 修改了代码文件
- 使用 `git add` 命令暂存了需要提交的更改

```bash
# 编辑文件后
git add .
# 或者添加特定文件
git add src/my-feature.js
```

### 2. 运行技能
在 Claude Code 中输入以下任一命令来触发技能：

```
/commit-code
```

或者简单地说：
- "提交我的更改"
- "生成提交信息"
- "总结代码变更"

### 3. 查看分析结果
技能会自动执行以下操作：
- 分析 Git 暂存区的更改
- 识别新增、修改、删除的文件
- 统计新增和删除的代码行数
- 按文件类型进行分类

### 4. 智能提交信息生成
技能会根据以下因素自动生成提交信息：

#### 提交类型识别
- `feat`: 新功能开发
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试代码相关
- `chore`: 日常维护任务

#### 作用域推断
根据文件路径自动推断作用域：
- `components`: src/components/*
- `utils`: src/utils/*
- `services`: src/services/*
- `pages`: src/pages/*
- `styles`: src/styles/*

## 高级功能

### 自定义提交信息
如果您不满意自动生成的提交信息，可以：
1. 拒绝自动生成的信息（输入 'N'）
2. 输入您自己的提交信息

### 更改预览
在提交前，技能会显示以下信息：
- 更改的文件列表
- 文件类型分类
- 新增/删除的代码行数
- 建议的提交信息

### 批量处理
对于包含多种类型更改的提交，技能会：
- 识别主要更改类型
- 提供综合性的提交信息
- 列出详细更改清单

## 仓库管理功能

### 1. 仓库初始化
如果当前目录不是 Git 仓库，技能会：
- 询问是否初始化 Git 仓库
- 创建基本的 .gitignore 文件
- 添加所有文件到暂存区
- 准备首次提交

### 2. 远程仓库管理
技能会检测是否已配置远程仓库：
- 如果没有配置，会提示设置远程仓库 URL
- 可以连接到 GitHub、GitLab 等平台
- 询问是否推送提交到远程仓库

### 3. 撤销仓库初始化
通过选择操作模式 3，可以撤销 Git 仓库初始化：
- 删除 .git 目录
- 保留所有项目文件
- 恢复到非 Git 状态

### 4. 删除整个仓库
通过选择操作模式 4，可以删除整个仓库：
- 安全删除仓库目录及所有内容
- 包含确认步骤防止误删
- 切换到父目录

### 5. 清理本地历史
通过选择操作模式 5，可以清理本地 Git 历史：
- 重置本地 Git 历史记录
- 保留远程仓库配置
- 重新初始化仓库但保持远程连接

### 6. 仓库状态检查
通过选择操作模式 6，可以检查仓库状态：
- 检查是否为 Git 仓库
- 显示远程仓库连接状态
- 显示当前分支信息
- 统计未提交和未推送的更改

## 操作模式详解

当运行 `/commit-code` 时，技能提供以下操作模式：

### 模式 1: 提交代码（默认）
- 智能分析代码更改
- 生成提交信息
- 执行提交操作
- 可选择推送至远程仓库

### 模式 2: 初始化仓库
- 创建新的 Git 仓库
- 生成 .gitignore 文件
- 添加所有文件到暂存区

### 模式 3: 撤销仓库初始化
- 删除 .git 目录
- 检查未提交更改并警告
- 需要用户确认

### 模式 4: 删除整个仓库
- 删除整个仓库目录
- 包含安全确认步骤
- 切换到父目录

### 模式 5: 清理本地 Git 历史
- 重置本地历史但保留远程配置
- 适用于需要清理提交历史的场景
- 保留远程仓库连接信息

### 模式 6: 检查仓库状态
- 显示详细的仓库信息
- 包括 Git 状态、远程连接、分支等
- 提供统计信息

### 模式 7: 设置远程仓库
- 配置远程仓库 URL
- 支持 GitHub、GitLab 等平台
- 不会自动推送现有更改

### 模式 8: 推送提交到远程仓库
- 将本地提交推送到远程仓库
- 检查远程连接状态
- 使用当前分支进行推送

## 参数和选项

当前版本支持以下交互选项：
- 确认自动生成的提交信息
- 输入自定义提交信息
- 取消提交操作
- 选择不同的操作模式
- 确认破坏性操作（如删除仓库）

## 最佳实践

### 通用最佳实践
1. **分批提交**：尽量保持每次提交的功能单一
2. **预先暂存**：使用 `git add` 明确指定要提交的更改
3. **审阅更改**：在最终提交前仔细检查技能提供的更改摘要
4. **遵循规范**：让技能帮助您生成符合 Conventional Commits 的信息

### 仓库管理最佳实践
1. **备份重要数据**：在执行删除或重置操作前备份重要数据
2. **确认操作**：仔细阅读确认提示，特别是删除相关操作
3. **定期同步**：经常推送更改到远程仓库以防数据丢失
4. **合理使用撤销功能**：在初始化错误或需要重新开始时使用撤销功能

## 故障排除

### 常见问题

**Q: 技能提示不是 Git 仓库**
A: 确保您在 Git 初始化的项目目录中，或选择初始化仓库选项

**Q: 没有找到任何更改**
A: 使用 `git add` 命令暂存您的更改后再运行技能

**Q: 生成的提交信息不符合预期**
A: 检查暂存的更改是否符合单一职责原则，或者手动输入自定义提交信息

**Q: 如何撤销仓库初始化？**
A: 运行技能并选择操作模式 3（撤销仓库初始化）

**Q: 如何删除整个仓库？**
A: 运行技能并选择操作模式 4（删除整个仓库），系统会要求确认

**Q: 如何清理 Git 历史？**
A: 运行技能并选择操作模式 5（清理本地 Git 历史），这将保留远程配置

### 错误处理

如果遇到错误，请检查：
1. 当前目录是否为有效的 Git 仓库（如果是需要的话）
2. 是否有权限读取 Git 仓库信息
3. Python 环境是否正常（需要 Python 3.7+）
4. 网络连接是否正常（如果是远程操作）

## 安全注意事项

- 技能在提交前会提供预览以供确认
- 不会自动提交可能包含敏感信息的更改
- 尊重项目的 .gitignore 设置
- 在执行破坏性操作前会要求用户确认
- 撤销初始化功能会警告可能的数据丢失
- 删除仓库功能包含多重确认步骤

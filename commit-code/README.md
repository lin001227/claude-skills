# commit-code 技能

## 概述

commit-code 是一个智能代码提交助手，专为简化 Git 提交流程而设计。它可以分析您的代码更改，自动总结变更内容，并生成符合 Conventional Commits 规范的提交信息。

## 功能特点

### 📊 Git 更改区域分析
- 自动扫描工作目录中的所有更改
- 区分已暂存和未暂存的更改
- 识别新增、修改、删除和重命名的文件
- 提取更改的关键信息进行摘要

### 🔍 文件类型识别
- 自动识别文件类型（新文件、更新文件、删除文件等）
- 根据文件扩展名确定编程语言
- 为不同类型的文件应用合适的处理策略

### 🧠 智能提交信息生成
- 根据变更内容自动生成符合 Conventional Commits 规范的提交信息
- 支持 feat、fix、docs、style、refactor、test、chore 等类型
- 提供清晰的提交信息结构（type(scope): description）

### 🛡️ 安全检查
- 在执行提交前进行预览
- 允许用户编辑生成的提交信息
- 检测并警告潜在的敏感信息泄露

### 🚀 仓库管理
- 自动检测是否为 Git 仓库
- 智能初始化新仓库（包括 .gitignore 配置）
- 连接远程仓库（GitHub、GitLab 等）
- 自动推送提交到远程仓库

### 🔄 仓库操作管理
- **撤销初始化**：可撤销 Git 仓库初始化，删除 .git 目录
- **清理历史**：清理本地 Git 历史记录同时保留远程配置
- **删除仓库**：安全删除整个仓库目录
- **状态检查**：详细检查仓库状态和配置

## 安装和配置

此技能已包含在 Claude Code 中，无需单独安装。

## 使用方法

### 基本用法

1. 准备您的更改：
   ```bash
   # 编辑文件后
   git add .
   # 或者添加特定文件
   git add src/my-feature.js
   ```

2. 运行技能：
   ```
   /commit-code
   ```

3. 技能将：
   - 分析您的更改
   - 生成更改摘要
   - 提供安全扫描
   - 建议提交信息
   - 询问您是否使用建议的提交信息
   - 如有需要，推送更改到远程仓库

### 处理新项目

如果在非 Git 仓库中运行技能：
1. 技能会检测到这不是 Git 仓库
2. 询问是否初始化新的 Git 仓库
3. 创建基础 .gitignore 文件
4. 添加所有文件到暂存区
5. 生成提交信息并提交
6. 询问是否连接远程仓库并推送

### 高级操作

技能支持多种操作模式：

1. **提交代码**：智能分析并生成提交信息
2. **初始化仓库**：创建新的 Git 仓库
3. **撤销初始化**：删除 .git 目录，恢复到非 Git 状态
4. **删除仓库**：安全删除整个仓库
5. **清理历史**：重置本地 Git 历史同时保留远程配置
6. **检查状态**：查看仓库详细信息
7. **设置远程**：配置远程仓库连接
8. **推送提交**：将提交推送到远程仓库

## 工作流程

1. **仓库检测**：检查当前目录是否为 Git 仓库
2. **初始化**：如需要，初始化 Git 仓库和配置
3. **检测更改**：扫描 Git 暂存区以识别所有更改
4. **分析内容**：确定更改的类型、影响范围和文件用途
5. **安全检查**：扫描可能的敏感信息泄露
6. **生成消息**：创建符合规范的提交信息
7. **执行提交**：使用最终确定的提交信息
8. **推送**：如需要，将提交推送到远程仓库

## 技术架构

此技能由以下几个模块组成：

- `main.py`：主入口点
- `main_enhanced.py`：增强版入口点，支持多种操作模式
- `enhanced_commit.py`：增强版主功能，包含仓库管理
- `file_type_detector.py`：文件类型分析和分类
- `security_checker.py`：安全风险检测
- `repo_manager.py`：仓库初始化、撤销和管理功能
- `repo_management_advanced.py`：高级仓库管理功能
- `SKILL.md`：技能定义和描述
- `config.json`：配置设置
- `USAGE.md`：使用说明

## 常见问题

**问：如果项目还没有 Git 仓库怎么办？**
答：技能会自动检测并询问是否初始化 Git 仓库，包括创建 .gitignore 和添加文件。

**问：如何推送提交到远程仓库？**
答：技能会检测是否有远程仓库连接，如没有会提示您添加，然后可选择推送。

**问：怎样撤销仓库初始化？**
答：运行技能并选择"撤销 Git 仓库初始化"选项，这将删除 .git 目录但保留您的文件。

**问：如何删除整个仓库？**
答：运行技能并选择"删除整个仓库"选项，系统会提示确认以防止误删。

**问：为什么我的提交被取消了？**
答：可能是因为安全检查发现了潜在的风险（如硬编码密钥），系统会询问您是否继续。

**问：如何自定义提交信息？**
答：在技能运行时，您可以选择编辑建议的消息或输入全新的自定义消息。

**问：技能支持哪些编程语言？**
答：支持超过 50 种主流编程语言和文件格式，包括 JavaScript、Python、Java、Go、Rust 等。

# commit-code 技能

## 概述

commit-code 是一个智能代码提交助手，它可以帮助用户总结Git更改区的代码变更，并自动生成符合约定式提交（Conventional Commits）规范的提交信息。该技能不仅能识别新增文件和更新文件，还能提供详细的变更摘要，帮助开发者快速、准确地提交代码。

## 触发条件

当用户表达以下意图时会自动触发此技能：

- "提交我的更改"
- "生成提交信息"
- "总结代码变更"
- "查看 Git 状态"
- "准备 Git 提交"
- "提交代码"
- "整理 Git 更改"
- "写提交信息"
- "commit message"
- 或任何与 Git 提交相关的请求

## 功能特性

### 1. Git 更改区域分析

- 自动扫描工作目录中的所有更改
- 区分已暂存和未暂存的更改
- 识别新增、修改、删除和重命名的文件
- 提取更改的关键信息进行摘要

### 2. 文件类型识别

- 自动识别文件类型（新文件、更新文件、删除文件等）
- 根据文件扩展名确定编程语言
- 为不同类型的文件应用合适的处理策略

### 3. 智能提交信息生成

- 根据变更内容自动生成符合 Conventional Commits 规范的提交信息
- 支持 feat、fix、docs、style、refactor、test、chore 等类型
- 提供清晰的提交信息结构（type(scope): description）

### 4. 变更摘要功能

- 提供简洁明了的变更列表
- 突出显示重要的变更部分
- 支持按文件类型或变更类型分类展示

### 5. 安全检查

- 在执行提交前进行预览
- 允许用户编辑生成的提交信息
- 避免意外提交敏感信息

## 使用方法

### 基本用法

1. 用户完成代码修改后，调用此技能
2. 技能自动分析当前 Git 状态
3. 生成变更摘要
4. 根据变更内容建议合适的提交信息
5. 用户可以确认或修改提交信息
6. 执行实际的 Git 提交操作

### 高级功能

- 支持选择性提交（仅提交部分更改）
- 提供多级别的提交信息详细程度
- 支持自定义提交信息模板
- 验证提交信息是否符合规范

## 适用场景

- 日常代码提交工作流
- 需要规范化提交信息的团队项目
- 提高代码提交效率的个人开发
- 遵循语义化版本控制的项目
- 需要自动化提交过程的持续集成环境

## 注意事项

- 此技能仅在 Git 仓库中运行
- 会尊重用户的 .gitignore 配置
- 执行提交前会提供预览以供确认
- 严格按照 Conventional Commits 规范生成提交信息

## 示例输出

### 变更摘要示例：

```
已修改文件:
- src/components/Button.jsx (新增组件)
- src/styles/main.css (样式更新)
- README.md (文档更新)

新增文件:
- src/utils/validation.js
- tests/unit/validation.test.js
```

### 生成的提交信息示例：

```
feat(components): 添加新的按钮组件

- 实现 Button 组件及其样式
- 添加输入验证功能
- 更新相关文档
```
